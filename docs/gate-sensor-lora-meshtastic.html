Gate Sensor — LoRa · Meshtastic · Solar UPS · Carrier PCB · Build Guide & BOM

BOM v2 · 1× sensor node · Heltec LoRa32 V3 · Arduino Nano · 433MHz RF · Waveshare Solar Manager D · 2× 18650 · Meshtastic private channel output · Custom carrier PCB — 75×66mm · KiCad · JLCPCB · AliExpress / Amazon sourcing
⚡ Architecture summary — A 433 MHz wireless door sensor broadcasts an RF code every open/close event. An Arduino Nano (always-on, ~20mA) receives the code via an RXB6 module and forwards plain text over UART to a Heltec LoRa32 V3 running Meshtastic firmware. Meshtastic relays the message to your private channel. A Waveshare Solar Power Manager D with 2× 18650 and a 6 V 2 W solar panel powers everything. No Xiaomi / BLE / Zigbee needed — a $3 433 MHz sensor does the same job without encryption headaches.

Phase 1 Flash Meshtastic on Heltec LoRa32 V3 ~15 min
1Flash Meshtastic firmware

Go to flasher.meshtastic.org in Chrome. Connect Heltec V3 via USB-C. Select:

  • Device: Heltec WiFi LoRa 32 (V3)
  • Version: Latest stable

Click Flash. The flasher erases and writes in one step. Device reboots into Meshtastic.

ℹ No Arduino IDE needed for this step. The web flasher handles everything including the bootloader.
2Add Heltec to the Meshtastic app

Open the Meshtastic phone app → + → Bluetooth. The Heltec will appear as Meshtastic_XXXX. Pair it. The OLED will show Connected.

3Configure frequency and private channel

In the app go to Radio Config → LoRa. Set region to match your country (EU_868, US_915, etc.).

Go to Channels. Edit Channel 0 (or add a new channel): set a Pre-Shared Key and a channel name (e.g. GATE). This becomes your private channel. Your receiving Meshtastic device must have the same channel config.

⚠ EU 868 MHz: Meshtastic defaults to 1% duty cycle compliance. With heartbeats every 5 minutes and event messages, you will stay within limits comfortably.
4Enable the Serial Module

Via Meshtastic app → Module Config → Serial, or via CLI:

# Using meshtastic Python CLI (pip install meshtastic)
meshtastic --set serial.enabled true
meshtastic --set serial.mode TEXT
meshtastic --set serial.rxd 47        # Heltec V3 default RXD
meshtastic --set serial.txd 43        # Heltec V3 default TXD
meshtastic --set serial.baud BAUD_9600
meshtastic --set serial.timeout 1000
meshtastic --set serial.channel 0     # Send to channel 0 (your private channel)
✓ Any text line terminated with \n sent to the Heltec UART at 9600 baud is broadcast as a channel message. No additional code runs on the Heltec.
Phase 2 Learn Your 433 MHz Sensor Codes ~10 min · do this before flashing the bridge code
1Install rc-switch library

In Arduino IDE: Sketch → Include Library → Manage Libraries → search rc-switch → Install.

2Upload the code-learning sketch
#include <RCSwitch.h>

RCSwitch rf = RCSwitch();

void setup() {
  Serial.begin(115200);
  rf.enableReceive(4); // RXB6 DATA → Nano D2 (INT0)
}

void loop() {
  if (rf.available()) {
    Serial.printf("Code: %lu  Bits: %d  Protocol: %d\n",
      rf.getReceivedValue(),
      rf.getReceivedBitlength(),
      rf.getReceivedProtocol());
    rf.resetAvailable();
  }
}
3Record open and close codes

Open Serial Monitor at 115200 baud. Open and close your sensor several times. You will see two distinct codes — note them both. Example output:

Code: 1234561  Bits: 24  Protocol: 1   ← gate OPEN
Code: 1234560  Bits: 24  Protocol: 1   ← gate CLOSED
ℹ The code repeats 3–5 times per event (RF retransmission). This is normal. You only need the unique value, not the duplicates.
Phase 3 Arduino Nano Bridge Code 433 MHz → UART → Meshtastic · with heartbeat
1Required libraries

Install via Arduino IDE Library Manager:

  • rc-switch — 433 MHz receive
  • SoftwareSerial — included with Arduino core, no install needed
2Full bridge sketch — paste your codes at the top
#include <RCSwitch.h>
#include <SoftwareSerial.h>

// ── Config ─────────────────────────────────────────
#define CODE_OPEN     1234561UL   // ← replace with your OPEN code
#define CODE_CLOSED   1234560UL   // ← replace with your CLOSED code
#define SENSOR_NAME   "Gate"

#define RF_PIN        2           // RXB6 DATA pin (now D2)

// Timing
#define HEARTBEAT_MS   300000UL  // 5 min heartbeat
#define DEBOUNCE_MS       200UL  // ignore retransmits

// ── Objects ────────────────────────────────────────
RCSwitch rf = RCSwitch();
SoftwareSerial meshSerial(4, 3); // RX=D4(unused), TX=D3 → Heltec GPIO47

// ── State ──────────────────────────────────────────
unsigned long lastHeartbeat = 0;
unsigned long lastRFEvent   = 0;

// ── Send to Meshtastic ──────────────────────────────
void sendMesh(String msg) {
  meshSerial.println(msg);
  Serial.println("TX: " + msg);
}

// ── Heartbeat ───────────────────────────────────────
void sendHeartbeat() {
  sendMesh("GATE NODE: Online");
}

// ── Setup ────────────────────────────────────────────
void setup() {
  Serial.begin(115200);
  meshSerial.begin(9600);
  rf.enableReceive(RF_PIN);
  delay(1000);
  sendHeartbeat(); // announce online at boot
}

// ── Loop ─────────────────────────────────────────────
void loop() {
  unsigned long now = millis();

  // ── 433MHz event ──────────────────────────────────
  if (rf.available()) {
    unsigned long code = rf.getReceivedValue();

    // debounce: ignore retransmits within 200ms
    if (now - lastRFEvent > DEBOUNCE_MS) {
      lastRFEvent = now;

      if (code == CODE_OPEN)   sendMesh(SENSOR_NAME + String(": OPEN"));
      if (code == CODE_CLOSED) sendMesh(SENSOR_NAME + String(": CLOSED"));
    }
    rf.resetAvailable();
  }

  // ── Heartbeat every 5 min ─────────────────────────
  if (now - lastHeartbeat >= HEARTBEAT_MS) {
    sendHeartbeat();
    lastHeartbeat = now;
  }
}
⚠ Board selection: In Arduino IDE select Arduino Nano · Processor: ATmega328P (Old Bootloader) for most cheap CH340 Nano clones.

Assembly Steps

1Drill and prep the box
  • Side wall: 12 mm hole for PG7 cable gland (solar panel cable)
  • Side wall: 16 mm hole for SMA bulkhead connector (LoRa antenna)
  • Deburr all holes. Fit cable gland and SMA bulkhead before populating boards.
2Mount boards on standoffs
  • Mark M3 hole positions for Nano and Heltec on enclosure floor
  • Drill 3.2 mm pilot holes, thread M3 brass standoffs (10 mm F/F)
  • Waveshare Solar Manager D has its own mounting holes — use same M3 standoffs
  • Heltec OLED should face upward toward the lid if you want to see status without opening
3Connect LoRa antenna

Connect Heltec's onboard IPEX/U.FL antenna port → short pigtail cable → SMA bulkhead on box wall → external antenna outside. The enclosure is ABS plastic so RF can pass through the wall, but a direct external antenna is still strongly recommended for range.

⚠ Never operate the Heltec's LoRa transmitter without an antenna connected. The SX1262 PA will be damaged by unmatched impedance.
4Route solar cable

Thread solar panel cable through PG7 cable gland. Connect + and − to the SOLAR+ / SOLAR− terminals on the Waveshare board. Tighten the cable gland seal. Apply a bead of silicone sealant around any remaining gap inside.

5Insert 18650 cells

Insert both cells into the Waveshare Solar Manager D slots — check polarity markings. Both cells should be the same brand and at approximately the same charge level. The board LED will indicate charging status.

6Solar panel mounting

Mount the solar panel on the enclosure lid, or on a nearby fence post with the cable routed to the box. South-facing (northern hemisphere) at a 30–45° tilt for best year-round generation. Secure with UV-resistant cable ties or silicone adhesive.

✓ Tip: Even in winter or overcast conditions, a 2 W panel produces enough charge to keep the system online indefinitely with normal gate activity.

3Review the generated board in KiCad

After the script runs, pan and zoom around the board. Check:

  • All three component outlines visible — Heltec socket (top-left), Nano socket (right), RXB6 (lower-center)
  • USB-C notch visible as a rectangular cutout at the top edge between the Heltec rows

4Run Design Rules Check (DRC)

In KiCad PCB Editor: Inspect → Design Rules Checker → Run DRC

Expected results on a freshly generated board:

DRC itemWhat to do
Unconnected items (ratsnest) Note which nets are unrouted. Most will be minor stubs — route them in Step 5. J4 will show one unconnected item between 5V_IN and 5V — this is intentional (the switch closes it externally).
Clearance violations Should be zero on a freshly generated board. If any appear, check that your DRC rules use standard 0.2 mm clearance. File → Board Setup → Design Rules.
Silk over pad Some silkscreen labels may overlap pads at the dense Heltec socket area. Acceptable for fabrication — fab houses ignore silk on pad. Click Ignore for these.
Footprint warnings The generated footprints use raw pads rather than library footprints. DRC may warn about missing 3D models or courtyard overlaps. Safe to ignore for fabrication.
5Route remaining ratsnest stubs

The script routes all key signal and power tracks. A few short stubs may remain depending on pad placement precision. Use the Interactive Router to close them:

  • Press X to activate the route track tool
  • Click the start pad of a ratsnest line, route to the destination pad, press Esc when done
  • Use U to select connected tracks and inspect routing
  • For GND stubs: route any remaining GND pads down to the horizontal GND bus at Y=64 mm
ℹ J4 ratsnest — the single unconnected line between J4 pin 1 (5V_IN) and J4 pin 2 (5V) should NOT be routed. The external switch closes this connection. Leave it as-is.
6Add GND copper pours (strongly recommended)

A GND pour on both layers floods all unused copper with ground, automatically connecting all GND pads and dramatically improving signal return paths and EMI immunity.

  1. F.Cu pour: Place → Add Rule Area → draw a rectangle covering the entire board (click corners, close with double-click)
  2. In the zone settings dialog: Layer = F.Cu · Net = GND · Clearance = 0.3 mm · Min width = 0.25 mm
  3. Repeat for B.Cu with Net = GND
  4. Press B to fill all zones — copper floods appear on both layers
  5. Re-run DRC to confirm no new violations from the pour
✓ Result: The board should now show very few or zero DRC errors and all GND connections resolved via the pour rather than individual tracks.

9Soldering guide — what goes where

When boards arrive, solder in this order (small components first, tallest last):

ComponentActionNotes
R1–R5, C1, C2 (passives) Solder direct Axial through-hole. Bend leads, insert, solder, trim flush. Verify values with multimeter before inserting.
J2 (screw terminal) Solder direct Push flat against PCB. Solder from below. Open screws face outward (toward board edge).
J4 (JST-PH 2-pin holes) Solder JST-PH through-hole pins or solder bridge See Step 10 — power switch options. Solder the right-angle or vertical JST-PH header into the holes.
U2 RXB6 socket Solder 1×4 male header onto RXB6, then solder RXB6+header direct into PCB holes If RXB6 has bare holes (no pins), solder a 1×4 male header onto it first. Then insert and solder the whole assembly into the board. This board is fixed — not removable.

10J4 power switch — three options

J4 is a 2-pin JST-PH (2.0 mm pitch) footprint inline on the 5V rail. Pin 1 is always-live from J2; pin 2 feeds the switched 5V bus to Heltec and Nano. Choose one of:

Option A — External rocker/toggle switch (recommended)
J2 (+5V) ──► PCB trace ──► J4 pin1 ──► wire ──► SWITCH ──► wire ──► J4 pin2 ──► 5V bus 1. Solder a right-angle JST-PH 2-pin header into J4 holes 2. Crimp a JST-PH female housing onto a 2-wire pigtail 3. Wire other end to a panel-mount rocker switch in the enclosure wall 4. Mount switch in 12mm hole in IP65 box side wall
Option B — Solder bridge (always on, no switch)
J4 pin1 ──► [solder blob between the two holes] ──► J4 pin2 Simply bridge the two J4 holes with a small blob of solder on the back of the board. System powers on as soon as J2 receives 5V from the Waveshare board.
Option C — JST-PH inline cable with switch (cleanest)
J4 ──► JST-PH female ──► cable ──► inline rocker ──► JST-PH male ──► (looped back to J4 pin2) Buy a pre-wired JST-PH inline power switch (common on RC / LiPo accessories). Plugs directly into J4 — no crimping needed. Search: "JST PH 2pin inline switch"
✓ Recommended: Option A with a mini rocker switch mounted in the enclosure wall. Lets you power-cycle without opening the box. Drill a 12 mm hole in the IP65 box side, snap in a waterproof rocker switch, and wire with 2× short JST-PH cables.
⚠ Known Gotchas & Troubleshooting
SymptomFix
Meshtastic receives no messages from UART Check that serial.enabled true and serial.mode TEXT are set in Meshtastic module config. Verify baud rate matches (9600 on both Nano and Meshtastic). Check that Nano TX → Heltec GPIO47 (not GPIO43 which is TX). Swap RX/TX if swapped.
RCSwitch receives no codes / garbage values RXB6 is on 5V — verify VCC (pins 4+5) is connected to the 5V bus. Check DATA pin (pin 7) is on Nano D2 (INT0). Try moving sensor closer during first test. Use a short wire antenna on the RXB6 ANT pin (17.3 cm for 433 MHz) for better range.
Same code fires repeatedly / debounce not working 433 MHz sensors retransmit 3–10 times per event. The DEBOUNCE_MS check in the loop suppresses duplicates. Increase to 500ms if still seeing doubles in Meshtastic.
Heltec not detected by Meshtastic app after flash Hold the Heltec BOOT button while pressing RST to force DFU mode, then reflash. Some V3 boards need a full erase first — in the web flasher check Full Erase before flashing.
No solar charging / Waveshare LED always red Verify solar panel is 6V (not 5V or 12V). Check polarity — reverse polarity protection exists but still confirm + / −. Measure panel voltage in sunlight: should be 5.5–6.5V open circuit. Shade or winter conditions may reduce to 3–4V — insufficient for MPPT.
Heltec OLED blank after Meshtastic flash Normal — Meshtastic dims the OLED after a timeout to save power. Press the button on the Heltec to wake it. OLED shows node info, channel, battery level.
Gate sensor code not recognized (wrong code) Re-run the code-learning sketch in Phase 2. The KERUI D026 uses KERUI encoding rather than plain EV1527 — rc-switch can still read it but the protocol number may differ. Make sure rc-switch captures a stable repeating code on open AND close separately. If only one code appears, the sensor may be open-only — switch to KERUI D026 or Sonoff DW2-RF which both send open+close. If Protocol 2 or 3 appears, add rf.setProtocol(2) before enableReceive.
Meshtastic duty cycle warning (EU) The heartbeat every 5 minutes plus event messages stay well within 1% duty cycle. If you reduce the heartbeat interval below 2 minutes and have a busy gate, you may hit limits. Increase HEARTBEAT_MS if warnings appear.
Water ingress in enclosure Ensure PG7 cable gland is fully tightened and O-ring is seated. Apply silicone sealant around the SMA bulkhead nut inside. Check lid gasket is clean and not pinched by wires. Face cable entry downward if possible so water runs away from the opening.
